Vex robotics truspeed simulation

Dublin High team 5327D - 2016/17

X Drive Simulation

Introduction

The images to the left are screenshots of a simulation that I created during my time with the Dublin High 5327D robotics team. The code below is a sample of this open source project I have worked on that demonstrates the differences of a linear acceleration of the wheels in an XDrive versus a logarithmic function that will accelerate based off the type of function.

  • Learn More

Overview

Overall, the judges very much liked my simulation, and we continued building on it and updating it. The end result gave me the ability to introduce and further explain the concept of truSpeed as well as provide answers to problems and questions we had ourselves while developing this robot. We discovered that along with a more realistic drive (like an actual automobile), the truSpeed functions I wrote provided better precision and accuracy for the controls. With a cubic function we were able to have a dead zone of about 20 units, which proved incredibly useful because our controller was not perfect and after some wear and tear the joystick began to give improper readings. However, we also were able to stop unnecessary strafing that was presented with the false values of the old controller, and we got the robot to be most efficient for moving in all directions, as well as turning and accelerating.

Elevator Project

Principles of Engineering - 2017

Project 3.2.

Introduction

The image to the left is the complete finished product of out elevator project. I was teamed with Michael Lee, Franklin Fitzpatrick, and Zachary Rowen. We each contributed to this project in our own way but my task was primarily building. As you can see it is built completely using Vex robotics parts, and was also programmed using the robotc code with an implementation of a PID loop for keeping the elevator at its specific level, along with distance based acceleration.

  • Look at our RobotC code!

Overview

Our task was simply to create a mechanism that would be able to transport people across different height levels in a building, using a cost efficient, and safe design. For our solution, we created an elevator that uses motor powered gears to climb up toothed beams along the sides of the walls. The motor powers the driver gears, which is in turn interlocked with an adjacent gear. The opposite turn of both of the gears directs the elevator upwards, or downwards, depending on the direction of the motor. Toothed beams line the sides of the walls, providing a track for the elevator to move up and down upon. Three buttons direct the movement of the elevator, and if a button is pressed, the elevator will move to its corresponding floor value. We have included a vex potentiometer sensor to read precise measurements for axle rotation and which is useful for determining elevator speed and current location.

Mechanical Winch

Principles of Engineering - 2016

Project 1.2.5

Introduction

The image to the left is the code that my team and I wrote for having a simulation for the winch device that we built. With my code here, we were able to simulate the winch with alternating variables such as the size and weight of the object, along with the input power being given to the winch. The goal for this project was to be able to determine the most optimal position and placement for our real life.

  • Try it out!

Overview

Our final prototype uses one Fischer Tek motor to power the mechanical winch. The motor is connected to a gearbox geared for speed, which in turn rotates an axle with a large gear. A smaller gear is meshed with the larger gear at a right angle in order to transform the horizontal rotational motion to vertical rotational motion. To stabilize the axle, we placed two pillars on either side of the motor. Our prototype lifted 180g of weight (1.764N) up 30cm (0.3m) in 5.82 seconds. The circuit drew 0.3A and had a voltage of 7.65V. The work done was 1.764N * 0.3m = 0.5292Joules. The output power was 0.5292 Joules/5.82 seconds = 0.091watts, and the input power was 0.30A*7.65V=2.295watts, making the efficiency 0.091 watts / 2.295 watts * 100%=4.0%.
